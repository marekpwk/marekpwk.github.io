Recursion

Basically recursion is a function that calls it self. For example:

Every time when the function calls itself, “n” is reduced by 1 and finally n will equal 0 and then it will return 1.

The idea of recursion is to transform a big problem into a smaller one, making it easier to solve. This is the most important part of recursion, when function will get to that part will finally return value and this is base case. It must be defined correctly, if not, the function will get into an infinite loop and will call itself forever.

It took me a lot of time to get a grasp on recursion. Once I did, I was applying recursion whenever it was possible, which is a mistake, because recursion is not an answer to everything.

Recursion makes code much cleaner, but it also makes it slower, so its up to the programmer to decide which is the best solution for his problem.
