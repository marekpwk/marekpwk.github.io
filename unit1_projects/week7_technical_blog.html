SQL vs NoSQL

What is the difference between SQL and noSQL databases?

First I want to explain what SQL means: Structured Query Language.

SQL database is a relational database which stores data in tables and columns. To use a relational database we need to create schema first which defines how our database will look like: tables , columns etc. We can only insert data that will match our schema. Anything else will be rejected. Data in a SQL are very well structured which allows you to query a database in very complicated ways. On the other hand, this becomes a scaling problem with huge databases. The answer to that is NoSQL.

As Wikipedia says “A NoSQL or Not Only SQL database provides a mechanism for storage and retrieval of data that is modeled in means other than the tabular relations used in relational databases.”

NoSQL doesn't need a schema, and data input can change over time. The structure is different then SQL and it allows you to get data much faster. Because of the lack of table structure, it doesn't allow work complicated queries like SQL does. To sum it up in a very simplified way, NoSQL has advantages when it comes to Big Data, when they need to be accessed very fast, and thats where SQL falls, because of its complicated structure.
